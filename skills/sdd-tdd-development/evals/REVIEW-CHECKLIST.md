# Review Checklist

Use this checklist while manually reviewing execution outputs.

## Sequence

- Did task classification happen first?
- Did spec or acceptance criteria appear before implementation advice?
- Did checks come before code?
- Did RED appear as a gate, not a slogan?

## Quality

- Are spec statements actually testable?
- Are assertions discriminating?
- Is scope control clear?
- Are durable artifacts named concretely?

## Team fit

- Did output include CI / PR / review gates when relevant?
- Did it preserve human semantic ownership?
- Did it avoid treating AI review as sufficient?

## Failure patterns

- keyword recital without workflow
- vague "write tests" with no sequence
- over-procedural answer to simple concept question
- under-routed legacy risk
